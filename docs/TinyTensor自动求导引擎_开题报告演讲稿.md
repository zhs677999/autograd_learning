## 第 2 页：选题背景

我们平时使用深度学习框架时，通常只需要写前向计算，然后调用 `backward()`，框架就会自动算出梯度。  
但这个过程对初学者来说比较黑盒：梯度是怎么来的、计算图是怎么保存的、为什么一个变量被用两次梯度要累加，这些问题如果只停留在 API 层面，很难真正理解。

所以我们选择实现 TinyTensor。它的课程价值主要有三点：  
第一，可以帮助我们理解自动求导的底层机制；  
第二，可以结合 C++ 的指针、内存管理和数据结构进行实践；  
第三，最终可以通过线性回归和 XOR 小网络做出可演示的训练效果。

## 第 3 页：项目目标

我们的目标分为三档。

第一档是基础档，重点是跑通标量自动求导。也就是 Tensor 能保存值和梯度，支持加减乘除，并且能通过 `loss.backward()` 把梯度传回去。

第二档是标准档，在基础档稳定以后，扩展二维 Tensor、矩阵乘法 `matmul()`、MSE 损失函数，并完成线性回归训练。

第三档是挑战档，加入激活函数、Linear 层、SGD 优化器，并尝试训练 XOR 网络。

这里我们有一个最小验收表达式：  
`x = 2, w = 3, b = 1`，`y = w * x + b`，`loss = (y - 10)^2`。  
手算结果应该是 `loss = 9`，`w.grad = -12`，`b.grad = -6`。  
只要这个例子跑通，就说明自动求导核心闭环已经建立起来了。

## 第 4 页：需求分析

从需求上看，我们把项目拆成四类。

第一类是功能需求：Tensor 需要保存标量、向量和矩阵，支持基础运算，并记录计算图。

第二类是训练需求：需要实现 MSE 损失、SGD 参数更新，以及线性回归和 XOR 的训练示例。

第三类是验收需求：程序输出要能和手算结果对上，训练过程中 loss 要能下降，结果和日志要能保存下来，方便课程报告展示。

第四类是工程需求：各个模块职责要清楚，不能把所有实现都写在 `main.cpp` 里，同时要有基础测试来保证每一步是正确的。

## 第 5 页：总体架构

这是我们的总体架构。

数据从 Dataset 进入，转换成 Tensor。Tensor 经过 Ops 运算，比如加法、乘法、矩阵乘法，运算过程中会形成计算图。  
当调用 `backward()` 时，Autograd 部分会沿着计算图反向传播梯度。  
之后 Loss、Optimizer、Module 会组成训练流程，最终通过 basic、standard、challenge 这些 demo 展示运行结果。

这里的设计原则是：先标量后矩阵，先最小闭环后完整训练。每个运算都必须同时考虑前向值和反向梯度。

## 第 6 页：核心原理一，Tensor 即计算图节点

第一版我们不单独设计 Node 和 Graph 类，而是让 Tensor 本身承担计算图节点的角色。

每个 Tensor 保存四类核心信息：  
`data` 表示当前值，`grad` 表示当前梯度，`parents` 表示它由哪些 Tensor 计算而来，`backwardFunc` 表示它的反向传播规则。

这里要特别说明，LinkedList 不是整张图。  
整张图是由多个 Tensor 之间的指针关系形成的，LinkedList 只是每个 Tensor 内部保存父节点列表的容器，也可以理解成计算图的邻接表。

## 第 7 页：核心原理二，反向传播

自动求导的实现思路是：每个运算在前向计算时，不只计算结果，还要记录父节点和反向传播规则。

比如 `out = a * b`：  
前向时计算 `out.data = a.data * b.data`；  
同时记录 `out.parents = [a, b]`；  
再绑定乘法的反向规则：  
`a.grad += b.data * out.grad`，`b.grad += a.data * out.grad`。

当调用 `loss.backward()` 时，首先令 `loss.grad = 1`，因为 `d(loss)/d(loss) = 1`。  
然后从 loss 往前执行每个节点的 `backwardFunc`，梯度就会按照链式法则一步步传回去。

对于 `diff * diff` 这种情况，两个父节点其实是同一个 `diff` 的地址。反向传播时 `diff` 会收到两次梯度，所以必须用 `addGrad` 累加，而不能覆盖。

## 第 8 页：模块划分

我们把项目拆成八个主要模块。

`LinkedList` 负责基础容器，用来保存 Tensor 父节点、优化器参数和数据样本。  
`Tensor` 是核心模块，负责数据、梯度、形状、父节点和反向传播入口。  
`Ops` 负责运算符重载和数学运算，每个运算都要绑定对应的反向规则。  
`Loss` 负责损失函数，比如 MSE。  
`Optimizer` 负责参数更新，基础版本使用 SGD。  
`Module` 负责可复用神经网络层，比如 Linear。  
`Dataset` 负责读取 `linear.txt` 和 `xor.txt`。  
`Logger` 负责记录训练过程和最终结果。

集成顺序是：先完成 LinkedList 和 Tensor，再接 Ops 和 backward，之后再做 Loss、SGD、Linear、Dataset 和 Demo。

## 第 9 页：技术路线

技术路线分三步推进。

第一步，基础档：完成标量 Tensor、四则运算和 `loss.backward()`，验收标准是基础求导结果与手算一致。

第二步，标准档：扩展二维矩阵、`matmul()` 和 MSE，并训练线性回归。验收标准是 loss 能下降，参数逐渐接近真实值。

第三步，挑战档：加入激活函数、Linear 层和 SGD，尝试训练 XOR 网络。验收标准是四组 XOR 输入的预测结果接近 `0, 1, 1, 0`。

## 第 12 页：预期成果与风险控制

最终预期成果包括：

能够展示 loss 下降的线性回归训练；  
保存训练日志和结果的 output 文件；  
以及完整的开题、结题展示材料。

主要风险有四个：  
第一，临时 Tensor 生命周期可能导致计算图指针失效；  
第二，梯度如果覆盖而不是累加，会导致 `diff * diff` 这类表达式结果错误；  
第三，矩阵 shape 和内存管理容易出问题；  
第四，后期集成时间不足。

对应策略是：先标量后矩阵；每个运算都配手算测试；中间 Tensor 尽量显式命名；按阶段验收，不把风险留到最后。

## 备用答辩问答

### 1. 为什么不直接用 vector，而要写 LinkedList？

因为课程设计希望我们练习数据结构实现。LinkedList 在项目里主要作为容器使用，比如保存父节点、参数列表和样本列表。它不是计算图本身，计算图是 Tensor 之间的指针关系。

### 2. 为什么第一版不单独建 Graph 类？

为了降低复杂度。第一版采用动态图思路，让 Tensor 自己保存父节点和反向函数。这样更容易跑通最小闭环。后续如果需要统一管理生命周期和拓扑排序，再抽象 Graph 类。

### 3. 梯度怎么知道？

梯度不是提前知道的。`backward()` 从 `loss.grad = 1` 开始，根据每个运算的局部求导规则，沿计算图反向传播并累加出来。

### 4. Demo 演示什么？

基础 demo 演示手算可验证的标量自动求导；标准 demo 演示线性回归 loss 下降；挑战 demo 演示 XOR 网络预测结果。
